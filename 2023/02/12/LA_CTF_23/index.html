So we can see format string bug in the highlighted line. Also note that the binary has Partial RELRO which means we can overwrite the GOT table in the memory.

Exploit

  • Stage 1 := Leak libc address with %39$p and also change main_called to 0 as well as puts GOT to main addresss to get another shot.
  • Stage 2 := Change these got to the following.
    • fgets -> gets ( for stack buffer overflow )
    • puts -> 0x04011B3 ( location in binary which calls fgets )
    • printf -> leave; ret gadget
  • Stage 3 := Stack overflow and then its just basic ret2libc.

Exploit

Looking at the source code we again have a format string bug in line number 12 and 9. This time itseems that we only have two chance.

  • First chance :- Leak libc address and LD address.
  • Second chance :- Overwrite to get shell!!!

But it wasn’t much useful. Then i used the same approach i used in my challenge gets. I created this challenge for Sekai CTF 2022.

Exploit

If you see the disassembly of fread on this part

   0x00007ffff7a9aa9d <+205>:  add    rsp,0x8
   0x00007ffff7a9aaa1 <+209>:  mov    rax,rbx
   0x00007ffff7a9aaa4 <+212>:  pop    rbx
   0x00007ffff7a9aaa5 <+213>:  pop    rbp
   0x00007ffff7a9aaa6 <+214>:  pop    r12
   0x00007ffff7a9aaa8 <+216>:  pop    r13
   0x00007ffff7a9aaaa <+218>:  ret

You can see there are some pop gadgets, The pop rbx and pop rbp are interesting. So my technique is to call fread while we stack pivot in the bss. And then call another fread just above our rsp in bss so that we can overwrite the values that are going to be popped on to control rbx and rbp register.
